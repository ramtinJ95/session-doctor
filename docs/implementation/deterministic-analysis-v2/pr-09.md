## Exact inputs and version selection

`analyze SESSION_ID` is an explicit write operation. Passive reads never create or repair rows.

1. Resolve `SESSION_ID` from immutable `normalized_entities` session payloads joined through
   normalization runs, bundles, and each bundle's primary snapshot. Require all matches to name
   one `(agent_name, logical_source_id)`; zero is unavailable and multiple logical sources are an
   identity conflict. Do not use mutable `sessions`/`session_sources` as the authority.
2. Select that logical source's greatest `capture_sequence`; do not order by timestamp, mutable
   projection insertion, content equality, or prior analysis. Require the latest capture's owning
   bundle, lifecycle observation, and normalized session. An unbundled/unparsed latest capture is
   unavailable; historical rows are never substituted.
3. Compute the current adapter capability-aware configuration hash. Require current normalization
   version/configuration. Prefer the exact current adapter version; otherwise choose the greatest
   parser-version key declared compatible. At the winning key, deduplicate repeated bundle links
   to the same run and require exactly one semantic `normalization_run_id`; competing identities
   are a conflict. Run ID is only a canonical sort key, never a semantic tie-break.
4. Load only that run's `normalized_entities` and `normalization_semantics`, plus the lifecycle
   observation for that exact capture. Verify ordering and lifecycle policy versions.
5. Create the core `analysis_identity` from the selected normalization/lifecycle IDs and versions.
   PR 9 uses `segmentation-v2`, stored ordering/lifecycle versions, and explicit
   `unavailable-pr09` sentinels for relation/result/finding/facet. The configuration hash covers
   `episode-analysis-v2`, `episode-anchor-v1`, `episode-membership-v1`, and
   `delegation-topology-v1`, never execution time.

Every topology counterpart goes through the same logical-source and greatest-capture algorithm as
the requested session. A witness proposes a counterpart but never selects its lifecycle. The
counterpart's unique latest primary capture must match the witness member's `snapshot_content_id`;
repeated equal-content captures therefore select greatest capture sequence and its lifecycle, not
an arbitrary equal-content observation.

Equal-content captures can reuse one normalization run. Their distinct lifecycle observations
still create distinct analysis identities. Stable content anchors allow episode/boundary IDs to
repeat under the composite analysis key. History is immutable until force-prune or an explicit
pre-1.0 derived rebuild.

### Historical retrieval

- Default `analyze SESSION_ID` selects/persists latest as above.
- `analyze SESSION_ID --snapshot-id ID` writes analysis for the bundle whose primary snapshot is
  `ID`. Require its immutable normalized session to equal `SESSION_ID`, then apply the same unique
  normalization-version selection to that exact bundle. Counterparts still use their unique latest
  captures and exact content handshake. This creates a new exact projection; it does not claim to
  recreate a prior multi-node lifecycle combination.
- `analyze SESSION_ID --projection-id ID` is read-only: require the stored projection's requested
  session to equal `SESSION_ID`, load only stored rows, and never recompute or substitute inputs.
- A missing/pruned ID is explicit unavailable. Snapshot/projection options are mutually exclusive.
  Exact historical replay is explicit normalization of that snapshot followed by
  `analyze --snapshot-id`; stored projection retrieval never depends on replay.

## Immutable anchor contract

Current adapter event IDs include source path and record index but not record content. They remain
normalized entity IDs; PR 9 must not treat them alone as immutable boundary anchors.

```text
analysis_anchor_id = hash("episode-anchor-v1", entity_kind, entity_id,
                          sha256(canonical normalized entity payload))
episode_id = hash("task-episode", segmentation_version, session_id,
                  first_user_analysis_anchor_id, last_user_analysis_anchor_id)
boundary_id = hash("episode-boundary", segmentation_version,
                   left_user_analysis_anchor_id, right_user_analysis_anchor_id)
```

Use the referenced raw event payload when a message has a source event; otherwise use the message
payload as an explicit fallback anchor. Persist anchor kind, normalized entity ID, payload digest,
and analysis anchor ID. Equal-content recaptures reuse anchors; an in-place record edit changes the
anchor even if path/index and adapter `event_id` stay equal. This changes persistence identity, not
PR 8's calibrated boundary-decision rules or frozen evaluation reference IDs.

## Delegation candidate discovery and exact handshake

Candidate discovery is separate from edge admission. Starting at the requested session:

- inspect complete captured topology witnesses for parent/child transcript members, sidecars,
  normalized parent status/candidates, and native spawn identity;
- close over deterministically witnessed parent and child candidates, including missing,
  ambiguous, mismatched, cyclic, and exact-counterpart-unavailable candidates;
- create a stable `topology_candidate_id` from policy version, sorted witness set, endpoint source
  identities/content-or-sentinel, direction, and native spawn identity-or-sentinel. It exists even
  when no child analysis does.

Retain all complete witnesses found in the closed selected bundles. For one endpoint pair, zero agreeing witnesses is unavailable;
disagreement on parent endpoint or spawn identity is ambiguous. Multiple witnesses agreeing on both
endpoints and spawn are one candidate with a stable-sorted witness set; none is chosen as "first."
Candidate/projection IDs hash the complete set, and binding provenance has one relational row per
witness bundle.

A candidate becomes a delegation binding only when one complete witness contains child transcript,
parent transcript, and required topology evidence, and:

- child metadata reports exactly one linked parent and native spawn/tool-use identity;
- selected parent/child primary `snapshot_content_id` values equal the corresponding witness
  transcript members;
- the parent normalization has exactly one matching native spawn tool call; and
- that call's source event has exactly one assigned parent episode.

Each node keeps its own lifecycle observation; no atomic directory snapshot is claimed. Missing
normalization/lifecycle, incomplete witness, duplicate spawn, absent parent episode, or unresolved
native topology persists candidate status and creates no binding. No latest-parent, nearest-time,
text, path, project, or model fallback is allowed.

Projection input order is `(role_rank, session_id, analysis_identity)` with ranks requested=0,

## Total membership projection

For each projection input, membership keys must equal—not merely count equal—the complete key set
`(analysis_identity, entity_kind, entity_id, entity_order)` from `normalized_entities`. This includes
session/source containers and parse warnings.

Assignment uses only source order and explicit normalized links:

1. Map each persisted event anchor to at most one source episode; duplicate ownership is fatal.
2. Raw events map by full event anchor. Message, tool call/result, command, file, and usage map first
   by `source_event_id` plus the referenced event payload digest.
3. Without direct evidence, follow only explicit links: tool result/command/file -> one tool call;
   message -> one native parent message; parse warning -> one `(source_id, record_index)`. One
   resulting episode assigns, multiple distinct episodes are ambiguous, none is unassigned.
4. Session/source containers use `container_not_episode_evidence`. Other unassigned reasons are
   `before_first_episode`, `no_episode_anchor`, `no_deterministic_anchor`, and
   `cross_source_partial_order`.

Timestamps never assign. Candidate episode keys sort by `(analysis_identity, episode_id)`.

### Membership/ownership truth table

```text
membership  source episode  candidates  owner status  owner full key  eligibility
assigned    present         empty       known         present         direct
assigned    present         empty       known         present         excluded_delegated
assigned    present         empty       unavailable   null            ineligible
ambiguous   null            nonempty    unavailable   null            ineligible
unassigned  null            empty       unavailable   null            ineligible
```

Checks enforce those nullability combinations, nonnegative entity order, and allowed enums;
transaction validation enforces nonempty canonical JSON candidates for ambiguous rows. Assigned
top-level rows own their source episode. Linked descendants own the top-level parent full
`(analysis_identity, episode_id)` and are excluded from additive aggregates. A known native child
with unresolved ancestry remains assigned to its source episode but has unavailable rollup owner
and is ineligible; it is never treated as a direct root.

## Delegation graph, ambiguity, cycles, and ordering

- A binding is child analysis -> one spawning parent analysis/episode. Every child episode gets
  one edge to that same parent. Binding uniqueness prevents episode-level parent divergence.
- Validate one parent per child and exact linked-status/edge completeness before insert and after
  reload. Every linked binding has one edge per child episode; no unresolved status has an edge.
- Detect strongly connected components before acceptance. Suppress all cycle edges, mark members
  cyclic candidates, and propagate unresolved ancestry to every descendant whose only path enters
  the cycle. Apply the same propagation from any ambiguous/unavailable ancestor frontier.
- Only a fully resolved acyclic path has a canonical top-level owner. Unresolved frontier and its
  descendants are rollup-owner unavailable and aggregate-ineligible.
- Evidence remains owned by its source episode. Parent projection never copies child rows. PR 9
  stores eligibility only; it computes no count or denominator.
- Parent and child source order stay separate. Edges expose spawn provenance but no merged ordinal,
  timestamp order, or visual timeline. Canonical graph serialization order
  `(depth, parent analysis+episode, child analysis+episode)` has no temporal meaning.

## Continuations and unavailable states

Native spawn evidence is PR 9's only cross-session attachment. Root sessions stay separate despite
similar text, project/model, paths, or timestamps. No continuation candidate, task-family identity,
similarity fingerprint, or independence claim is created; those remain unavailable until the
versioned relation framework owns them.

Supported-but-unresolved topology returns successfully with candidate statuses and ineligible
memberships. Missing requested exact input fails `EpisodeAnalysisUnavailable`. A missing candidate
counterpart is represented by `topology_candidate_id` without inventing an analysis identity.
Adapters declaring topology unsupported/unknown get explicit unavailable statuses for native-child
candidates; sessions with no child evidence get `not_child`/direct behavior.

## Public payload and deterministic ordering

`episode-analysis-v2` is:

```text
EpisodeAnalysisPayload
  schema_version = episode-analysis-v2
  requested_session_id
  analysis_identity                 # requested input
  episode_projection_id
  exact_inputs[]                    # run/bundle/lifecycle/policy IDs, role
  episodes[]                        # requested analysis only, with user/event/boundary IDs
  boundaries[]                      # requested analysis only, full anchors/evidence
  observations[]                    # requested analysis only
  memberships[]                     # requested analysis only, full ownership state
  delegation { candidates[], bindings[], child_episode_edges[] } # whole projection
```

Exact inputs order by projection input order; requested episodes/boundaries/observations by stored
ordinal then ID; memberships by normalized `(entity_kind, entity_order, entity_id)`; candidates and
graph rows by their canonical keys. Terminal mode prints requested identity/projection, episode and
boundary summaries, membership status counts, and each delegation status/edge—never an aggregate.

Latest analysis inserts then reloads this model from stored rows inside the same write transaction,
validates it, commits, and renders the loaded object. Historical retrieval uses one read snapshot.
Thus rendering cannot race a post-commit force-prune and never returns an ephemeral computation.

## Transaction, concurrency, replay, and pruning

1. Open one write transaction and select all exact inputs/witnesses from that transaction snapshot.
2. Compute immutable anchors, segmentation rows, exact membership key sets, candidate closure,
   accepted bindings, unresolved-ancestry propagation, and canonical ownership in memory.
3. Validate full keys, membership truth table, one-parent rule, cycle/status propagation, linked
   edge completeness, source-event uniqueness, and exact-input compatibility.
4. Insert semantic analysis, episode, projection, topology, and membership rows with conflict-ignore;
   compare every semantic column canonically. Exclude only first-write `created_at`, `started_at`,
   `completed_at`, and operational metadata from identity comparison. A mismatch rolls back.
5. Reload/validate the public model inside the transaction, then commit and render it.

DuckDB's single-writer boundary is load-bearing: overlapping writers begin selection only after
obtaining a write transaction. Identical/overlapping analyses serialize and the later writer must
pass idempotent comparison. A database busy/open error is explicit and leaves no partial rows;
there is no stale-input retry. Tests cover identical and overlapping topology writers.

Snapshot dependency inspection reports every affected analysis/projection ID and new-table count.
Ordinary prune blocks. Forced prune has two separate closures:

1. Delete every projection touching a pruned analysis **or directly citing the pruned bundle as a
   candidate/binding witness**, even if no analysis used that bundle as a primary input. Delete
   membership, child-episode edges, binding witnesses/bindings, candidate witnesses/candidates,
   inputs, then projection row.
2. Delete only analyses whose own lifecycle/normalization depends on the pruned snapshot: observation
   evidence, observations, boundary evidence and episode-boundary links, boundaries, event/user
   anchors, episodes, then semantic run. Analyses whose exact inputs survive remain available for
   other projections.

Then existing normalization/snapshot closure runs. A projection shared by surviving analyses is
removed, not truncated; those analyses can form a new projection only via explicit analyze. Existing
evaluation partial-corpus protections and post-commit checkpoint behavior remain.

## Adversarial, permutation, and metamorphic test matrix

### Exact input, identity, persistence, public contract

- Persist/reload every table/model; assert canonical `episode-analysis-v2` terminal/JSON round-trip.
- Reanalyze twice: same semantic IDs/rows/output; operational timestamps are not rewritten/compared.
- A/B/A latest selection uses final A lifecycle; unparsed latest C is unavailable while
  `--projection-id` reads stored A without recomputation.
- `--snapshot-id` writes an older exact target without latest substitution. Repeated equal-content
  counterpart captures select greatest sequence/lifecycle; multiple agreeing witnesses are all
  retained, while conflicting witnesses are ambiguous independent of row order.
- Same bytes active -> settled yield distinct analyses but equal content anchors/episode IDs. An
  in-place record edit at equal path/index changes anchor, episode, and boundary IDs.
- Exact parser wins; compatible unique fallback wins; incompatible or duplicate semantic winner
  fails. Duplicate links to one run do not conflict.
- Permute normalized rows, SQL insertion, candidate discovery, siblings, and timestamps: identities,
  role order, canonical JSON, and output remain equal.
- Parent-initiated versus child-initiated projection has distinct request-qualified identity and
  reconstructs exact roles without collision.

### Membership state space

- Full membership key set equals normalized entity key set for every input/projection.
- Positive and negative cases cover raw event; message direct/parent fallback; tool call; tool
  result direct/call fallback; command direct/call fallback; file direct/call fallback; usage
  direct; parse warning record lookup/null index; session/source containers.
- Equal record indexes in different sources do not correlate. Before-first-user, no-user,
  orphan/missing link, duplicate cross-episode candidates, and cross-source partial order cover all
  explicit reasons and truth-table rows.
- Removing a direct anchor degrades only through explicit link -> ambiguous/unassigned; never nearby
  timestamp. Timestamp permutations do not change membership; authoritative source-order changes do.

### Delegation and topology

- Root -> child -> grandchild gives exact provenance, one binding per child analysis, one edge per
  child episode, separate source order, top-level composite owner, and child exclusion.
- Multiple child episodes share one spawn parent; a spawn in parent episode two maps there.
- Same child bytes with changed parent bytes, missing sidecar/counterpart, incomplete witness,
  duplicate spawn, absent parent episode, unsupported capability, and native
  ambiguous/mismatched/missing statuses create no accepted binding and no direct eligibility.
- Cycle, cycle-with-descendant, and unresolved-ancestor-with-descendant propagate unavailable owner
  and ineligibility through the affected subtree. Removing the bad frontier restores the same clean
  graph as direct construction (metamorphic).
- Each entity has one row; no schema/payload exposes merged parent/child order or aggregate.

### Destructive, concurrency, packaging

- Similar independent roots produce no continuation/family ID.
- Ordinary prune reports/blocks all dependencies. Forced prune removes whole projections but only
  analysis rows whose own input is pruned; pruning a witness-only bundle also removes every citing
  projection. Shared surviving analyses/history and unrelated durable bytes remain.
- Injected failures at each table family and semantic conflicts roll back all PR 9 rows.
- Concurrent identical and overlapping-topology analyze calls serialize to valid idempotent state or
  one explicit database-busy failure, never partial/mixed rows.
- Fresh schema, v10 derived rebuild, and clean built-wheel install with `PYTHONPATH` cleared run the
  installed `session-doctor analyze` in terminal/JSON and historical modes. Full required gate passes.
